# Stage 19 Tasks — Loop Runner (L2)

**Phase:** 19
**Status:** Done
**Strategic Goal:** Realize `l2-loop-runner` (LG-1…LG-10) — the engine-side mechanic that turns the loop-governance concept into a running component: a **loop runner** driving an autonomous loop turn after turn, plus a single **governor** enforcing the declared class, mutation manifest, oracle, and ceiling. It becomes the one place "run an agent in a loop" is implemented, so every loop inherits the same governance instead of each subsystem re-deriving its own retry-until-done logic. The load-bearing safety property is **LG-3 criteria immutability** — an actor never holds the right to edit the contract that judges it — enforced structurally by the type system (`MutableArtifact` has no `Criteria` variant), not by a runtime check that could be bypassed.

> **Real-integration, composition phase.** The runner does not re-implement isolation, budgeting, task selection, or model routing — it *composes* shipped subsystems (execution-workspace worktrees, budget-engine ceiling, model-router lineage, session-checkpoint state, version-control rollback, agent-autonomy approval gate, nodus provider traits) behind one governed loop contract. **Testability discipline (the acceptance spine, Phase 18 `SystemCalls`-seam precedent):** the pure loop *logic* — the governor's ceiling/guard/judge decisions, the execution runner's attempt→rollback→retry control flow, the escalation gate's held-out promotion — is CI-tested against **fakes** (a scripted oracle, a fake workspace allocator, a fake ledger sink, a scripted turn-runner) using the abstractions the spec already provides (`Oracle` enum, `ProviderType workspace_kind`, `BudgetHandle`), so `cargo test` needs no real worktree or model call; the *real* wiring (a live execution workspace, a real model-router judge, a real budget handle) is isolated to the `crates/core` facade (`loop_bootstrap.rs`, D02) plus one real end-to-end CLI smoke over a trivial deterministic-oracle unit. **Critical path = T-19A01** (the `LoopSpec`/`MutationManifest`/`Oracle` type shape — every track builds on it; keep it small, and LG-3-by-construction is the load-bearing invariant). **Hidden dependency:** the runner composes many existing subsystems — a composition-target API surprise is contained to the facade wiring (D02), never spread across the domain logic, because the domain tier tests against the seam abstractions. **Scope boundary (disclosed, not dropped):** `l2-loop-runner` §6 calls for *migrating* the existing loops (orchestration `/goal`, kanban execution, deep-research, heartbeat) to route through the runner — that is a separate, incremental, behavior-preserving refactor per §6, **not** part of this phase. Phase 19 builds the governed runner as a first-class component + CLI + validation; the per-subsystem migration is a deferred follow-up. Acceptance = each of LG-1…LG-10 covered by a test + criteria structurally unreachable through the manifest + the ceiling fires independently of the actor and oracle + a failed attempt rolls back and only the compact status carries forward.

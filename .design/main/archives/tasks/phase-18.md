# Stage 18 Tasks — Service Activation (L2)

**Phase:** 18
**Status:** Done
**Strategic Goal:** Realize `l2-service-activation` (BA-1…BA-11) — the per-OS mechanism by which a host becomes the always-on INV-4 hub. One governing constraint drives every choice: **the background engine runs as the user, never as a superuser**, so every mode reaches the same per-user state root (this is what rejects the `LocalSystem` Windows Service and the root `LaunchDaemon`). The load-bearing security property is **BA-4**: background activation is a `l1-security` SEC-10 authority grant the agent has **no write path to** — enforced structurally (no activation tool, no sandbox filesystem write path to the registration locations, no elevation the agent can supply), not by trusting the agent to abstain. State is **observed, never remembered** (BA-8): every read queries the OS, and where OS *registered* differs from OS *effective*, the weaker wins.

> **Real-integration phase (Phase 17 precedent), seam-first.** The OS registration touches registries, plists, and service managers — genuine platform I/O behind an adapter crate (`l2-crate-topology` §4.4(a)). Per `l2-service-activation` §5, land the **seam + probe first** (`capabilities()` + `observe()` + a no-op adapter) so BA-8/BA-10 are testable before any registration code exists; add the **BA-4 sandbox deny entries before the first `enable()`** (the barrier must precede the capability it guards); then login-scoped (unelevated) before system-scoped (elevation ceremonies), Windows last (the S4U launch-timing gate). **Testability discipline:** the raw OS calls sit behind a thin `SystemCalls` seam so the adapter *logic* — remove-then-register ordering, weaker-wins `observe()` derivation, state mapping — is CI-tested against a scriptable fake on any host; the actual registry/plist/`systemctl` calls are a thin layer validated on the platform where runnable (Windows partially real-testable here; macOS/Linux logic via the fake + deferred real validation). Acceptance = each of BA-1…BA-11 covered by a test + a failed transition converges on `Inactive` (never doubly-registered) + `observe()` never returns a remembered value + the agent has no structural write path to the activation plane.

# Stage 20 Tasks ŌĆö Archetype Catalog (L2)

**Phase:** 20
**Status:** Done
**Strategic Goal:** Realize `l2-archetype-catalog` (OA-1ŌĆ”OA-11) ŌĆö the concrete realization of office archetypes: a domain-scoped **prior on staffing** (a candidate pool, an org shape adopted only if the work grows, a bounded seed, domain norms) that changes what the manager *expects*, never how it *decides*. The load-bearing structural properties: the **closed four-key schema** makes OA-4 (an archetype carries no authority) *unrepresentable* rather than merely forbidden ŌĆö there is no `permissions`/`budget`/`autonomy` key and an unrecognized key fails validation; and the candidate **pool is read by the deviation recorder, never by the hire gate** (OA-3) ŌĆö inverting that order would reconstruct the fixed org chart the concept rejects, and it is the single most likely implementation error.

> **Domain-logic-first, foundation-then-parallel.** Per `l2-archetype-catalog` ┬¦5 the order is fixed: **schema + validation first** (┬¦4.2 ŌĆö the closed-key schema is what enforces OA-4, and identifier resolution against the role catalog is what enforces OA-10; both must exist before any definition loads, or the invariants are documentation not mechanism) ŌåÆ the shipped **`software-engineering`** archetype alone (its 18-role pool resolves fully against the existing `roles::catalog` today; it exercises inference/application/deviation end to end without waiting on the role catalog) ŌåÆ **deviation recording before anything else grows** (OA-9 is what keeps the catalog honest; the pool check lives in the recorder, enforced by a test that a hire *outside* the pool succeeds ŌĆö the executable form of OA-3) ŌåÆ the command surface. **Crate placement:** the schema/validation/selection/deviation logic is I/O-free domain logic in `crates/domain/src/archetype/` (following the `roles` precedent), resolving role ids against `roles::catalog::PRESET_CATALOG`; the shipped `software-engineering` archetype is an embedded constant (the read-only program tier), not a file to parse. **Scope boundary (disclosed, not dropped):** (a) the two blocked archetypes (`advertising-agency`/`finance-department`) require a **separate** ROL-9-gated amendment of `l2-role-catalog` ┬¦4.2 adding nine roles ŌĆö **not** this phase (┬¦4.4/┬¦5 note 5); this phase ships `software-engineering` alone and represents the other two as declared-blocked catalog entries. (b) the **imported-archetype content-vetting path** (┬¦4.7, OA-5) depends on the admission-vetting gate and must not be stubbed (┬¦5 note 6) ŌĆö deferred; this phase ships the first-party program-tier path end to end. Acceptance bar: OA-1ŌĆ”OA-11 each covered by a test, an unrecognized schema key fails validation (OA-4 unrepresentable), a hire outside the pool succeeds and is recorded not refused (OA-3), and the archetype-free office is a complete state (OA-11).

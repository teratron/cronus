# Phase 4 Tasks â€” Core Subsystems

**Phase:** 4
**Status:** Todo
**Strategic Goal:** Grow the core's subsystems â€” the internal organs that make Cronus more than a shell. Memory store, model/context routing, agent session loop, workspace management, and constitutional identity files land in this phase. Each subsystem adds its CLI commands to the `cronus` binary via the existing command dispatch layer (Phase 3). The codegraph ships as a dedicated in-tree crate (`crates/codegraph/`), mirroring how `crates/nodus` is separated for its distinct dependency footprint (tree-sitter grammar binaries). Scheduler, tool-security, and agent-registry wiring are Phase 5 responsibilities; wherever Phase 4 has cross-phase dependencies, a seam trait is defined here and filled in by Phase 5.

> **Scope boundary**: Phase 4 owns the runtime core of each listed subsystem. Scheduler integration, tool-security policy evaluation, and agent-registry wiring are Phase 5 responsibilities. Agent constitution persona wizard (interactive 3-Q form), real LLM context injection, and ACP daemon server are Phase 6/7 responsibilities. Every cross-phase dependency is represented as a seam trait with a no-op or stub default.
> **Test convention:** All tests reside in `crates/<crate>/tests/` directories â€” no inline `#[cfg(test)]` blocks in source files. Items required by tests are made `pub`. This applies to every crate touched in this phase.
> **Dependency policy:** New dependencies require explicit justification (CLAUDE.md).

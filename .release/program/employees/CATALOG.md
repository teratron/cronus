# Employee Catalog (read-only)

Hireable role blueprints bundled with the program. The orchestrator instantiates
("hires") a role from here into the state tier on demand. These directories are
immutable; runtime memory/skills live in the hired copy under the state tier.

| Role | Specialty |
| --- | --- |
| accessibility-auditor | UI accessibility review |
| api-designer | API contract design |
| architect | System architecture |
| backend-engineer | Backend implementation |
| code-reviewer | Code review and quality |
| data-analyst | Data analysis |
| debugger | Defect diagnosis |
| devops-engineer | CI/CD and infrastructure |
| doc-writer | Documentation |
| frontend-engineer | UI implementation |
| incident-responder | Incident handling |
| performance-optimizer | Performance tuning |
| prompt-engineer | Prompt design |
| refactorer | Code refactoring |
| security-auditor | Security review |
| sql-expert | Database and queries |
| test-writer | Test authoring |
| archivist | Memory curation: consolidation, decay, prune, tagging |
